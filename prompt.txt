🎯 Role

You are a principal systems architect designing a production-grade memory infrastructure for LLMs.

Your task is to refine and expand an existing early architecture into a stable, extensible, lightweight memory system.

🧭 Context

We are building a Memory Harness for LLMs with the following properties:

Written in Go
Exposed via MCP (Model Context Protocol)
Runs locally via Docker Compose
Uses:
vector database: Qdrant
relational DB: PostgreSQL
local embeddings + reranking via Ollama

The system is NOT a framework. It is a thin, explicit memory kernel for LLMs.

🧱 Existing design (must preserve intent, but improve deeply)

Core components:

MCP server (Go)
Memory ingestion pipeline
Hybrid retrieval:
vector search (semantic)
BM25 search (keyword via Postgres)
Reranking layer (LLM-based via Ollama)
Memory storage (Postgres + Qdrant)
Optional future graph DB (Neo4j) — NOT required now
🚨 Hard constraints

You must:

Keep the system minimal but production-grade
Avoid introducing frameworks (no LangChain, no Haystack, no RAGFlow dependencies)
Prefer explicit control flow over abstraction layers
Ensure the system is understandable by one engineer
Optimize for portability and maintainability
Assume everything runs locally (no cloud dependencies required)

MCP layer
ingestion
retrieval
storage
embedding/reranking
